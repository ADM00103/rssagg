# IT News Aggregator

PHP-агрегатор IT-новостей с RSS-парсингом, извлечением полного текста, локальным кэшем картинок, сохранением в `txt`, вторичным `index.json`, автоматической группировкой новостей и отдельной RSS-лентой.

## Возможности

- Парсинг новостей из RSS.
- Догрузка полного текста со страницы новости.
- Извлечение и локальное сохранение первой картинки.
- Кэш каждой новости в отдельном `.txt`.
- Вторичный индекс в `index.json`.
- Очистка кэша по расписанию.
- Светлая и тёмная темы.
- Вывод карточками в стиле Telegram.
- Отдельная RSS-лента для подключение в агрегаторы.

## Требования

- **PHP 8.3**
- Расширения:
  - `curl`
  - `dom`
  - `SimpleXML`
  - `xml`
  - `mbstring`
- Включённый доступ на запись в:
  - `cache/txt`
  - `cache/img`
  - `data`
  - `logs`

> Важно: на старых версиях PHP может появляться ошибка 500. Проект рассчитан на **PHP 8.3**, на более старых версиях стабильная работа не гарантируется.

## Установка

### 1. Загрузить файлы на хостинг

Скопируй все файлы проекта в папку сайта, например:

```text
/public_html/news
```

### 2. Создать папки

Если папки не создались автоматически, создай вручную:

```text
cache/txt
cache/img
data
logs
```

### 3. Проверить права

Убедись, что веб-сервер может записывать файлы в папки `cache`, `data` и `logs`.

### 4. Настроить `config.php`

Открой `config.php` и укажи:

- свой домен в `site_url`;
- свой `cron_token`;
- список RSS-источников при необходимости.

Пример:

```php
'site_url' => 'https://w99522bk.beget.tech',
'cron_token' => 'YOUR_SECRET_TOKEN',
```

### 5. Запустить первичное обновление

Открой в браузере:

```text
https://w99522bk.beget.tech/refresh.php?token=YOUR_SECRET_TOKEN
```

После этого создадутся:
- `data/index.json`;
- `data/last_update.txt`;
- файлы кэша в `cache/txt`;
- кэш изображений в `cache/img`.

### 6. Открыть сайт

Главная страница:

```text
https://w99522bk.beget.tech/
```

RSS-лента:

```text
https://w99522bk.beget.tech/rss.php
```

или через rewrite:

```text
https://w99522bk.beget.tech/rss.xml
https://w99522bk.beget.tech/feed.xml
```

## Cron

Проект использует два cron-задания:

- `refresh.php` — обновление новостей;
- `cleanup.php` — очистка кэша и устаревших файлов.

### Вариант 1. Локальный cron на хостинге

Это предпочтительный способ.

Пример:

```cron
*/15 * * * * /usr/bin/php8.3 /home/USERNAME/public_html/news/refresh.php >/dev/null 2>&1
5 0 * * * /usr/bin/php8.3 /home/USERNAME/public_html/news/cleanup.php >/dev/null 2>&1
```

Если у хостинга другой путь к PHP, используй свой реальный путь к PHP-CLI.

### Вариант 2. Внешний cron-сервис

Если используешь внешний сервис вроде cron-job.org, вызывай URL с токеном:

```text
https://w99522bk.beget.tech/refresh.php?token=YOUR_SECRET_TOKEN
https://w99522bk.beget.tech/cleanup.php?token=YOUR_SECRET_TOKEN
```

> Иногда внешний cron-сервис может получать ошибку `403 Forbidden` или `connection failed` из-за ограничений хостинга, фильтров безопасности, HTTPS-редиректов или блокировки внешних запросов. В таком случае лучше использовать локальный cron через панель хостинга.

## Как работает кэш

Каждая новость сохраняется:
- в отдельный `.txt` файл;
- в `index.json`;
- в локальный кэш картинок, если изображение найдено.

Это ускоряет загрузку сайта и уменьшает нагрузку на источники.

## RSS

RSS-лента отдаёт:
- `title`
- `link`
- `pubDate`
- `description` с кратким текстом
- `content:encoded` с полным текстом
- `enclosure` для изображения, если оно есть

## Темы

На сайте есть:
- тёмная тема;
- светлая тема.

Выбор темы сохраняется в cookie.

## .htaccess

`.htaccess` нужен для:
- отключения листинга каталогов;
- работы `rss.xml` и `feed.xml`;
- базовой защиты служебных файлов;
- заголовков безопасности.

Если на хостинге не работает `mod_rewrite`, сайт всё равно откроется через `rss.php`.

## Возможные проблемы

### Ошибка 500

Проверь:
- что используется **PHP 8.3**;
- что включены расширения `curl`, `dom`, `SimpleXML`, `xml`, `mbstring`;
- что папки `cache`, `data`, `logs` доступны на запись.

### Неправильная кодировка

Если видишь текст вроде `Ð¼Ð¸Ð½ÑƒÑ‚Ñ…`, значит проблема с кодировкой или старым кэшем.

Что сделать:
- удалить `data/index.json`;
- удалить содержимое `cache/txt`;
- удалить содержимое `cache/img`;
- снова открыть `refresh.php`.

### Cron не срабатывает

Если внешний cron-сервис даёт ошибку:
- проверь URL;
- проверь токен;
- проверь HTTPS;
- попробуй локальный cron на хостинге.

## Быстрый запуск

1. Залить файлы.
2. Создать папки.
3. Указать `site_url` и `cron_token`.
4. Запустить `refresh.php`.
5. Проверить главную страницу.
6. Настроить `cron`.
