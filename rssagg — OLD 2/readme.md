# IT News Aggregator

Простой PHP-агрегатор IT-новостей с поддержкой RSS и парсинга страниц, локальным кэшем картинок, сохранением новостей в `txt`, вторичным индексом `index.json`, автоматической группировкой новостей и отдельной RSS-лентой для внешних агрегаторов.

## Возможности

- Парсинг новостей из RSS-лент.
- Догрузка текста и изображения со страницы новости.
- Сохранение каждой новости в отдельный `txt`-файл как кэш.
- Хранение вторичного индекса в `index.json`.
- Локальный кэш изображений.
- Автоматическая группировка похожих новостей.
- Светлая и тёмная тема.
- Карточный вывод новостей в стиле Telegram.
- Отдельная RSS-лента для подключения в RSS-ридеры и агрегаторы.
- Очистка `txt`, `img` и `index.json` по расписанию через `cron`.

## Требования

- **PHP 8.3**
- Расширения PHP:
  - `curl`
  - `dom`
  - `SimpleXML`
  - `xml`
  - `mbstring`
- Доступ на запись в папки:
  - `cache/txt`
  - `cache/img`
  - `data`
  - `logs`

> Важно: на более старых версиях PHP у проекта может появляться ошибка 500. Рекомендуется использовать именно PHP 8.3.

## Установка

### 1. Залей файлы на хостинг

Скопируй все файлы проекта в папку сайта, например:

```text
/public_html/news
```

### 2. Создай необходимые папки

Если они не создались автоматически, создай вручную:

```text
cache/txt
cache/img
data
logs
```

### 3. Проверь права доступа

Папки `cache`, `data` и `logs` должны быть доступны для записи веб-сервером.

### 4. Укажи свой домен

Открой `config.php` и измени:

```php
'site_url' => 'https://example.com/news',
```

на свой реальный адрес.

### 5. Запусти первичное обновление

Открой в браузере:

```text
https://your-site.com/news/refresh.php
```

После этого создастся `index.json`, кэш новостей и кэш изображений.

### 6. Открой сайт

Главная страница:

```text
https://your-site.com/news/
```

RSS-лента:

```text
https://your-site.com/news/rss.php
```

или, если включён rewrite:

```text
https://your-site.com/news/rss.xml
https://your-site.com/news/feed.xml
```

## Настройка cron

Проект использует `cron` для двух задач:

1. Обновление новостей.
2. Очистка кэша и старых данных.

### Пример cron-заданий

Открой crontab:

```bash
crontab -e
```

Добавь строки:

```cron
*/15 * * * * /usr/bin/php /var/www/news/refresh.php >/dev/null 2>&1
5 0 * * * /usr/bin/php /var/www/news/cleanup.php >/dev/null 2>&1
```

### Что делает cron

- `refresh.php` — запускается каждые 15 минут и обновляет новости.
- `cleanup.php` — запускается раз в сутки и удаляет старые:
  - `.txt` кэши новостей,
  - локальные изображения,
  - `index.json`, если он устарел.

> Если путь к PHP отличается на хостинге, замени `/usr/bin/php` на актуальный путь, который выдаёт твой хостинг.

## Как работает кэш

Каждая новость сохраняется:
- в отдельный `.txt` файл;
- в общий `index.json`;
- при наличии изображения — в `cache/img`.

Это ускоряет загрузку сайта и уменьшает нагрузку на внешние источники.

## Формат RSS

RSS-лента отдаёт:
- `title`
- `link`
- `pubDate`
- `description` с кратким текстом
- `content:encoded` с полным текстом новости
- `enclosure` для изображения, если оно есть

Лента совместима с большинством RSS-ридеров и агрегаторов.

## Темы оформления

На сайте доступно две темы:

- тёмная;
- светлая.

Переключение темы сохраняется в cookie браузера.

## .htaccess

Файл `.htaccess` нужен для:

- отключения листинга каталогов;
- переадресации `rss.xml` и `feed.xml` на `rss.php`;
- базовой защиты служебных файлов;
- добавления простых HTTP-заголовков безопасности.

Если у хостинга не включён `mod_rewrite`, сайт всё равно будет работать через прямые ссылки на `rss.php`.

## Устранение проблем

### Ошибка 500

Если видишь ошибку 500, проверь:

- версию PHP — должна быть **8.3**;
- наличие расширений `dom`, `SimpleXML`, `xml`, `curl`, `mbstring`;
- права на папки `cache`, `data`, `logs`.

Это означает проблему кодировки. Обычно помогает:

- очистка старого `index.json`;
- удаление старого кэша в `cache/txt` и `cache/img`;
- повторный запуск `refresh.php`.

### RSS не открывается

Проверь:
- существует ли `rss.php`;
- правильно ли настроен `.htaccess`;
- поддерживает ли хостинг `mod_rewrite`.

## Быстрый запуск

1. Залить файлы на хостинг.
2. Создать папки `cache/txt`, `cache/img`, `data`, `logs`.
3. Убедиться, что стоит PHP 8.3.
4. Запустить `refresh.php`.
5. Открыть сайт.
6. Добавить `cron` для автообновления и очистки.
